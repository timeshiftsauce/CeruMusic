# 更新日志 (Changelog)

> 本文件记录 Ceru Music (澜音) 每个发布版本的主要改动,主要面向 **终端用户** 视角:
> 写明新功能 / 修复 / 入口路径,而非提交级技术细节。
>
> 文件被 [`.github/workflows/main.yml`](.github/workflows/main.yml) 在打 tag 发布时
> 提取 **第一个 `## [vX.Y.Z]` 块** 作为 GitHub Release 正文,也就是软件内
> "检查更新" 弹窗显示的 **更新说明** 内容。所以发版前请把新版本写在文件顶部。
>
> 格式参考 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/),
> 版本号遵循 [Semantic Versioning](https://semver.org/lang/zh-CN/)。

## [v1.14.0] - 2026-05-15 · 听歌更省心,聊天更带感 ✨

本次更新带来响度均衡、歌词偏移、歌词复制面板,以及一起听的聊天 @ 提及与设置面板,
另外修复并打磨了不少播放与通知细节。

### 🎚 响度均衡 (Loudness Normalization)
跨曲音量自动拉齐,告别下一首突然变大/变小的尴尬。
- **入口**: 设置 → 音频效果 → 响度均衡 (可开关 / 选择参考响度)

### 🎼 歌词偏移 (Per-song Offset)
每首歌可以单独微调歌词时间轴,自动持久化保存。最近调过的 200 首会保留 90 天,
超出自动清理,不占空间。
- **入口**: 全屏播放页 → 右上角 ⋯ 更多菜单 → 歌词偏移
- 支持 ±100ms / ±500ms 预设,以及自定义毫秒数输入

### 📋 歌词复制面板 (Lyric Copy Overlay)
全屏歌词可以批量复制,支持选择是否带时间戳 / 翻译 / 音译。
- **入口**: 全屏播放页 → 右键点击歌词区域
- 或 全屏播放页 → 更多菜单 → 复制歌词

### 💬 一起听 · 聊天 @ 提及
聊天里输入 `@` 可以呼出成员列表,被提及的人会高亮显示并收到单独通知,
长消息再多也不会漏看队友 cue 你。
- **入口**: 一起听房间 → 聊天面板 → 输入 `@`

### ⚙️ 一起听 · 设置面板
房主和成员可以各自决定要不要新消息通知、弹幕、音效。
- **入口**: 一起听房间 → 右上角设置图标

### 🔔 通知收敛与体验优化
- 新消息通知改为右下角 toast 节流弹出,不再频繁打断
- 通知提示音统一,音量更柔和
- 全屏播放页收起时新消息走 toast 而非弹窗
- 聊天分享卡片支持单曲弹动作面板、歌单直接跳页

### ✨ 其它打磨
- 聊天面板使用虚拟滚动 (virtua),长聊天记录滚动更顺滑
- 聊天面板与歌词复制面板的滚动条样式统一
- 一起听单曲卡片重做深色样式,角色感知更清晰
- 队列 (PendingPanel) 支持多人点歌头像堆叠展示

## [v1.11.2] - 2026-05-08 · 差分更新 & 主题跟随系统

### ✨ 新增
- **增量更新 (差分下载)** —— 借助 electron-updater blockmap 协议,只下载新旧版本的字节级差异,
  180MB 升级包通常只需几 MB 流量
- **更新方式选择对话框** —— 远端 release 包含 blockmap 时可在"差分更新(推荐)/全量更新"间手动切换
- **差分失败自动回退全量** —— 触发 error 时主进程自动降级到全量下载,并 toast 通知
- **主题跟随系统** —— 主题选择器支持"跟随系统亮/暗",自动同步
- **基于 Cloudflare Workers 的更新服务器** —— 边缘缓存替代进程内缓存,自动合并多架构 macOS 描述
- **Cloudflare Workers 反向代理** —— 流量经 CF 全球网络透传,解决国内访问 GitHub 不通问题,
  支持 Range 请求差分下载
- **手动构建并上传到指定 Release 的 workflow** —— 支持补传 / 替换已发布 Release 中的产物

### 🔧 优化
- 更新错误码转换为用户友好中文提示,统一主进程与渲染进程接口
- 网络层移除 axios 改用 fetch,加入超时/重试,DNS 失败时走 DoH (Cloudflare 1.1.1.1) 兜底
- 路由预加载策略加入用户交互感知和优先级排序,移除欢迎页版本获取超时限制
- 多视图 CSS 性能优化,移除不必要的 GPU 合成层和 content-visibility

### 🐛 修复
- 修复版本号错误回退问题
- 修复 mac intel/arm64 runner 都构建双架构的问题 (改用显式 `target:arch` 钉死架构,构建时间减半)
- 修复 CI cleanup 规则误删 `*.blockmap` 产物,保证 release 包含差分更新元数据
- 更新服务从 `update.ceru.shiqianjiang.cn` 迁移到 `update.cerumusic.top`
